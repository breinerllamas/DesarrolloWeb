Múltiples resoluciones

Los llamados media queries son una serie de funciones que incorpora CSS3 para que las páginas web se adapten o adopten una maquetación diferente según la resolución en la que se muestre la página, ya sea en navegadores sobre ordenadores con grandes monitores, tablets, móviles o incluso displays de neveras, yogurteras, coches o lavadoras conectadas a Internet.

Existe una muy amplia variedad de resoluciones de pantalla. Para intentar agruparlas en grupos estándares podemos utilizar los siguientes rangos:

  • Resoluciones iguales o menores de 480px: para móviles.
  • Resoluciones de 481px a 768px: para tablets.
  • Resoluciones mayores de 768px: para ordenadores.

¿Dónde colocarlos?

Como cada página HTML tiene su propia estructura y necesita una adaptación personalizada, este código CSS se suele añadir en el <style> del <head>, dentro de la propia página HTML, pero siempre al final, justo antes de cerrar la etiqueta </style>.

<head>
  <style type="text/css">

    body {
      background-color: white;
      font-size: 14px;
    }

    #bloque1 {
      font-size: 15px;
    }

    /* ← Primero todos los selectores normales */

    /* ← Después los mediaqueries, siempre al final */
    @media only screen and (max-width: 320px) {
      body {
        background-color: red;
      }
      #bloque1 {
        font-size: 12px;
      }
    }

  </style>
</head>

Primero se colocan todos los selectores habituales de la página (ya sea en el <head> o en un archivo CSS externo) y después de todos ellos se colocan los mediaqueries con sus selectores CSS dentro.

Funcionamiento de los Mediaqueries

Un mediaquerie agrupa diferentes selectores CSS e identifica en qué condiciones deben ejecutarse. Normalmente estas condiciones están relacionadas con la resolución horizontal del dispositivo donde se visualiza la página.

En el ejemplo anterior @media only screen and (max-width: 320px) se indica que si la resolución horizontal es menor de 320 píxeles se cambiará el color de fondo a rojo y el texto del #bloque1 pasará a 12px. Si no se cumple esa condición, se aplican los selectores CSS normales (fondo blanco y 14px).

Sintaxis

Tenemos varias posibilidades según el rango que queramos cubrir:

max-width — resolución menor que la indicada

@media only screen and (max-width: 320px) {
  /* Se ejecuta cuando la resolución sea menor de 320px */
}

min-width — resolución mayor que la indicada

@media only screen and (min-width: 980px) {
  /* Se ejecuta cuando la resolución sea mayor de 980px */
}

Los dos parámetros clave son:

  • max-width: "cuando la resolución horizontal sea menor que la indicada".
  • min-width: "cuando la resolución horizontal sea mayor que la indicada".

Combinando dos rangos

También se pueden combinar min-width y max-width para ajustar un rango concreto de resolución:

@media only screen and (min-width: 320px) and (max-width: 480px) {
  /* Se ejecuta solo entre 320px y 480px de resolución */
}

Exclusivo para móviles y tablets

En un móvil no existe una ventana del navegador que se pueda redimensionar: toda la pantalla es la ventana. Además, cuando se visualiza una página en un móvil, el contenido se escala para mostrarse al completo, por lo que generalmente la página aparece muy pequeña. Para resolver estos dos problemas existen dos soluciones:

Usar device-width en los mediaqueries

Añadiendo device hacemos referencia a la anchura real del dispositivo y no al ancho de la ventana del navegador:

@media only screen and (min-device-width: 320px) and (max-device-width: 480px) {
  /* Solo para dispositivos con ancho real entre 320 y 480px */
}

El metadata viewport

Para que el contenido no se escale automáticamente debemos añadir el siguiente <meta> dentro del <head>:

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=2">
  • width=device-width: el contenido se adapta a la resolución horizontal real del dispositivo.
  • initial-scale=1: escala inicial del 100% (con "2" sería el doble; se admiten decimales como "0.75" o "1.25").
  • maximum-scale=2: zoom máximo que podrá hacer el usuario (en este caso el doble del tamaño original).
  • minimum-scale=1: impide que el usuario reduzca el contenido por debajo del tamaño original.

Comparativa de resultados en móvil

❌ Sin mediaqueries — Sin viewport

La página se escala automáticamente para visualizarse entera, reduciendo todo el contenido aunque cupiera perfectamente en la pantalla del dispositivo.

⚠️ Con mediaqueries sin device — Sin viewport

Se visualiza exactamente igual que sin mediaqueries. Sin viewport, usar mediaqueries sin device en móviles y tablets carece de utilidad.

⚠️ Con mediaqueries con device — Sin viewport

Los mediaqueries funcionan correctamente, pero la página se muestra a escala muy reducida al no tener viewport, para permitir ver el contenido completo.

✅ Con mediaqueries — Con viewport

Para que los mediaqueries funcionen correctamente es imprescindible el uso del metadata viewport con width=device-width e initial-scale=1, independientemente del uso o no de device en los mediaqueries.
