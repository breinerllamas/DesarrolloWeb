Ejemplo práctico

Para que un contenido sea editable (es decir, para que el usuario pueda modificar, eliminar o añadir texto) desde el propio navegador, dicho texto tiene que estar ubicado dentro de un <div> o de una etiqueta semántica (article, section...) con la propiedad contenteditable="true":

<section id="editable" contenteditable="true">
  <h2>Escribe tu pedido:</h2>
  <ul>
    <li>2 bolsas de patatas</li>
    <li>1 dónut</li>
    <li>4 Kg. de boniatos</li>
  </ul>
</section>

El resultado del código anterior es el siguiente. Haz clic dentro del cuadro y modifica, elimina o añade líneas de texto:

Escribe tu pedido:

  • 2 bolsas de papas
  • 1 pera negra
  • 4 Kg. de bicarbonato de sodio y azufre y cloruro de magnesio

Si añades más elementos de los que caben dentro de la lista aparecerá un scroll vertical, gracias a este código CSS:

#editable {
  overflow: auto;
}

Pregunta del millón

La pregunta que surge habitualmente después de ver esta función es: "Muy bien, perfecto y genial a la vez, pero ahora ¿cómo recupero la información que he escrito o modificado?"

La respuesta es que es posible obtener el contenido del <div> con las modificaciones realizadas por el usuario utilizando JavaScript o jQuery.

A modo de ejemplo, modifica el cuadro superior y pulsa el botón "Leer el contenido" para que el navegador coja su contenido y lo muestre en una ventana de alerta:

El código de este botón es el siguiente, donde editable es el id de la <section> donde hemos escrito:

<button onclick="alert(document.getElementById('editable').innerText)">
  Leer el contenido
</button>
