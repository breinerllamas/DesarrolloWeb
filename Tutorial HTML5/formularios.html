Formularios en HTML5

Los formularios fueron una de las primeras mejoras que se dieron a conocer de HTML5, sobre todo por su gran capacidad de autovalidación de manera extremadamente fácil y sin necesidad de utilizar JavaScript. Todas las etiquetas del formulario deben estar ubicadas dentro de <form> y </form>.

En el atributo action se indica la ruta del fichero (por ejemplo un .php) que procesará los datos, y con method se elige el método de envío. El método post garantiza mayor seguridad para los datos enviados.

<form action="enviarmail.php" method="post"> ... </form>

Ejemplo práctico

A continuación se muestra un formulario de registro que utiliza las principales etiquetas nuevas de HTML5. Observa cómo algunos campos son obligatorios (borde rojo/azul según el estado), cómo el correo electrónico valida su formato automáticamente y cómo el primer campo recibe el foco al cargar la página gracias a autofocus.

Preferencias

Primero vamos a ver algunas de las características de HTML5 que igual has pasado por alto:

  • De entrada, el cursor podría está situado por defecto dentro del primer campo, para que el usuario pueda escribir sin tener que hacer clic en él.
  • Si sitúas el cursor encima de cualquier campo podrás leer una explicación adicional sobre esa pregunta.
  • El segundo y tercer campo son obligatorios, por lo que -cuando estés en ellos- tendrán el borde rojo hasta que sean rellenados y de manera correcta.
  • El correo electrónico, además sólo aceptará el valor introducido si detecta una @ enmedio y un formato apropiado.
  • Además, el correo electrónico y otros campos ofrecen un valor de ejemplo que se elimina automáticamente al introducir cualquier valor.
  • Al pulsar el botón "Enviar los datos" sin haber introducido los campos obligatorios o bien el correo electrónico con un formato no válido aparecerá un mensaje de alerta con información adicional que informa de los errores encontrados.

Tipos de <input> más usados

text

Permite introducir texto normal. Con autofocus el cursor se sitúa automáticamente en el campo al cargar la página. Con placeholder se muestra un texto de ejemplo en gris que desaparece al escribir.

<input type="text" autofocus placeholder="Tu nombre" />

email

Valida automáticamente que el valor introducido contenga una @ y un formato de dominio apropiado. Muy útil combinado con required.

<input type="email" required placeholder="usuario@dominio.com" />

password

Enmascara automáticamente los caracteres que escribe el usuario con círculos, protegiendo la privacidad de la contraseña.

<input type="password" required />

date

Muestra un selector de calendario para que el usuario introduzca una fecha. Con value se especifica la fecha inicial por defecto.

<input type="date" value="2000-01-01" />

range

Genera una barra deslizante (slider). Con min, max y step se controlan los límites e incrementos del valor.

<input type="range" min="1" max="7" step="1" value="2" />

color

Muestra una paleta de colores para que el usuario seleccione un color. Con value se indica el color inicial en hexadecimal.

<input type="color" value="#FF0000" />

number

Permite introducir valores numéricos con botones + / −. Se controla el rango válido con min y max.

<input type="number" min="0" max="9000" step="1" value="10" />

radio y checkbox

radio: permite elegir solo una opción del grupo (todos deben tener el mismo name). checkbox: permite seleccionar varias opciones a la vez (cada uno tiene un name diferente). Con checked se marca una opción por defecto.

<input type="radio" name="sexo" value="mujer" checked /> Mujer
<input type="checkbox" name="dest1" value="colombia" checked /> Colombia

file

Permite seleccionar un fichero del disco duro del usuario. Con accept se puede filtrar el tipo de archivo permitido.

<input type="file" accept="image/*" />

submit y reset

submit envía los datos del formulario a la dirección indicada en action. reset limpia todos los campos volviendo a los valores iniciales. El texto del botón se define con value.

<input type="submit" value="Enviar los datos" />
<input type="reset" value="Limpiar formulario" />

Atributos importantes

  • required: El campo es obligatorio; sin rellenarlo no se envía el formulario.
  • autofocus: El cursor se sitúa en este campo al cargar la página (solo uno por formulario).
  • placeholder: Muestra un texto de ejemplo en gris que desaparece al escribir.
  • title: Texto informativo que aparece al pasar el cursor por encima del campo.
  • autocomplete: Permite el autocompletado del campo según entradas anteriores del usuario.
  • value: Valor inicial del campo al cargar la página.
  • name: Identificador del campo que se envía al servidor junto con su valor.

Estilos CSS y autovalidación visual

Con CSS3 podemos indicar visualmente el estado de cada campo usando pseudoselectores:

  • input[required]: aplica estilos a los campos obligatorios.
  • input:invalid: aplica estilos cuando el valor introducido no es válido.
  • input:valid: aplica estilos cuando el valor es correcto.
input[required] { border-color: blue; box-shadow: 0 0 5px blue; }
input:invalid { border-color: red; box-shadow: 0 0 5px red; }
input:valid { border-color: green; box-shadow: 0 0 5px green; }
